Asynchronous Programming - 第十一章:Task.WhenAll

Created2025-10-21|Updated2025-10-22|未来よりの返歌
|Post Views:

你叫三個朋友(小弟)幫你去不同的店買東西(跑腿)：

小明去買雞蛋
小華去買牛奶
小美去買麵包

你不會等小明回來才讓小華去，你會同時叫他們出門。然後你說：「等你們全部都回來，我們再一起做早餐！」這就是 Task.WhenAll ，等全部人回來，再開始做接下來的事。
它的本質是，把很多 Task 包裝成「一個新的 Task」，這個新 Task 會在所有子 Task 都完成後才完成。(to execute multiple asynchronous tasks concurrently，and wait for all of them to complete)


🎵 簡單實驗 - 什麼時候觸發了任務執行?

👉 可以看到，Task started 都發生在 All tasks added! 之前，代表任務在被「建立（呼叫）」時就開始跑了，不等 WhenAll 才啟動



🎵 下載檔案預覽

流程大概是這樣子的 : 「給一串網址 → 同時下載所有文字檔 → 全部下載完成後印出內容預覽」

  1. 建立多個任務，DownloadFileAsync(url) 會立即啟動下載（非同步），不會卡住主線程。所以這段 foreach 很快就會跑完。
  2. Task.WhenAll 等待全部完成，當你呼叫 await Task.WhenAll(getContentTasks) 時，程式會同時等所有下載任務完成，等全部都完成後，才會回傳所有下載結果。
  3. 處理結果，回傳的 contents 是一個「陣列」，裡面包含每個下載結果（byte[]）。

如果改成這樣

  1. 先等第一個下載完
  2. 再下載第二個
  3. 一個一個排隊

這樣速度會慢很多，因為網路下載可以同時進行，沒必要等一個完成再開始下一個。所以 Task.WhenAll 幫你「並行處理任務」可以極大化效能



🎵 Validators

假設你現在有一個「付款流程」需要做多項驗證 (Validation)，
例如：

AValidator：檢查交易紀錄是否存在
BValidator：檢查金額是否合理
CValidator：檢查使用者身份是否合法

每個驗證都要去查資料庫或呼叫外部 API，這些動作通常是「IO-bound」（例如讀寫資料、網路請求）而這類任務最適合用非同步 (async/await) 執行

因為三個任務幾乎同時啟動，也幾乎同時結束，即使只用一個執行緒，也能達到「同時在處理」的效果，就是所謂的「Concurrency（並行）」，但是要小心執行的任務如果本身無法一次處理這麼多的 Request 可能會較容易失敗

如果任務是「自己在 CPU 裡算東西」，程式不會釋放執行緒，不管你用多少個 Task，這些運算都要「真的佔用 CPU 時間」，若你用 Task.WhenAll，系統可能會開多個執行緒同時計算 →
結果是 CPU 爆滿、效能下降，在這種情況下，除非你有多核心 CPU 且希望平行計算（Parallelism），否則應該就用同步執行（foreach），因為 CPU-bound 的任務在 async 之下沒有加速效果。

🎵 WaitAll 與 WhenAll

項目 Task.WaitAll() Task.WhenAll()
類型 同步 API 非同步 API
是否阻塞 Thread ✅ 會阻塞目前執行緒 ❌ 不會阻塞（await 等待）
回傳型態 void Task 或 Task<T[]>
可否用 await ❌ 不行 ✅ 可以
適合用在哪裡 傳統同步程式（Main, Console） 非同步程式（async 方法）
若任務拋出例外 直接在 WaitAll 時丟出 會包成 AggregateException 在 await 時丟出

你在等三個朋友（小明、小華、小美）買早餐

  • Task.WaitAll → 你站在門口發呆，不幹別的，一直等三個人回來（整個人「卡住」）
  • Task.WhenAll → 你先交代他們去買早餐，然後自己去煮水、準備餐具。三個都回來後自動通知你。（你「非阻塞等待」）

展現阻塞的差異

Task.WhenAll

三個 DoWork 任務同時啟動；

但主執行緒仍然持續印出 主線程還在工作中…；

主線程沒有被卡住；

等待 2 秒後，任務都完成，才印出最後一句。

👉 這就是「非阻塞」。

Task.WaitAll

Task.WaitAll 這行一執行後，主執行緒就「停住」；會發現 2 秒內主線程什麼都不做；直到三個任務都結束才印出 Main 繼續執行



🎵 結語

就這樣，小明、小華、小美拿著雞蛋、牛奶、麵包陸續回來了。

「欸，你們怎麼都同時回來？是不是偷偷約好的？」你懷疑地看著他們。

小明得意地說：「我們用了 WhenAll 啊！不像你之前用 WaitAll，站在門口像個門神一樣動也不動。」
小華接著說：「對啊，我們出門的時候你還在煮水、準備餐具，一點都沒浪費時間。」
小美冷血的笑著補充：「而且如果我們其中一個人迷路（拋例外），其他人還是會繼續買東西，不會全部卡住。」

你看著桌上豐盛的早餐材料，心想：「程式碼如人生，與其苦苦等待，不如學會並行。讓任務像朋友一樣各自努力，最後一起享受成果。」

「那我們開始做早餐吧！」
「等下，誰要負責洗碗？」
「…這個用 Task.Run(洗碗機) 吧」

🥞 Task.WhenAll：讓等待變成一種藝術，讓並行成為一種默契。
