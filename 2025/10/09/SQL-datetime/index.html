SQL - 時間序列の鏡

不要在查詢條件裡對時間欄位做函式運算，要讓 SQL Server 能用索引！並且適時使用暫存表（Temp Table）或 CTE 讓時間運算更有效率!

🔹 一般寫法

  • GROUP BY CONVERT(CHAR(7), TradesOrderGroup_DateTime, 120)與 ORDER BY CONVERT(CHAR(7), TradesOrderGroup_DateTime, 120) 都會讓 SQL Server 對每筆資料進行字串轉換
  • CONVERT() 出現在 SELECT、GROUP BY ORDER BY，SQL Server 會視為三次運算，CPU 多做了三倍的事

🔹 優化寫法

  • 用暫存表先篩出「今年的資料」
  • 後續的分組與轉換只在後面上運算
  • 用 CTE 預先轉換一次月份 CONVERT() 只執行一次

💼 平均出貨處理天數



💼 總結

  • Non-SARGable 是效能殺手，當條件中使用函式（例如 DAY(), CONVERT()）作用在欄位上，SQL 無法用索引，只能全表掃描

  • 時間範圍比較是最佳實踐用 >= 起始時間 AND < 結束時間 方式查詢，最能讓 SQL Server 使用索引

  • 盡量減少重複運算，函式放在多處（如 SELECT、GROUP BY、ORDER BY）會多次執行 → 改成先計算一次（CTE 或暫存表），效能更好

  • 暫存表是查詢優化的好幫手，尤其在多層次查詢或報表邏輯中，可以先篩資料、再做聚合，減輕主查詢壓力
