WebApi - Swagger

Created2025-09-25|Updated2025-09-25|WebAPI
|Post Views:

套件

Swashbuckle.AspNetCore.SwaggerUi

文件

https://swagger.io/api-hub/

註冊

是在 ASP.NET Core 專案裡，啟用 Swagger (OpenAPI) 文件產生器的註冊。

AddSwaggerGen() 來自 Swashbuckle.AspNetCore 套件。它會把你的 API Controllers / Actions 掃描起來，自動產生符合 OpenAPI 規範 (Swagger) 的 API 文件。這份文件通常位於 /swagger/v1/swagger.json。

launchsettings.json

Swagger 與 OpenAPI 的關係

Swagger 是一個「專案 / 工具集」

最早由 Wordnik 在 2010 左右提出的 API 文件產生工具，叫做 Swagger。

它的核心想法：用一份 規格文件 (JSON / YAML) 來描述 API，然後可以自動產生「文件頁面」和「測試工具」。

這份規格文件，後來被業界廣泛接受。

OpenAPI 是「規範 (specification)」

到了 2016 年，Swagger 規格捐給了 Linux Foundation，成立了 OpenAPI Initiative (OAI)。

從那之後，Swagger 規格改名為 OpenAPI Specification (OAS)。

也就是說：

Swagger 規格 1.x / 2.0 → 叫 Swagger
Swagger 規格 3.0+ → 正式名稱是 OpenAPI

工具與標準的區分

OpenAPI：是「標準」，只是一份 JSON/YAML 規格，描述 API 的結構。
Swagger：是一組工具，幫你讀取/產生/展示 OpenAPI 文件。

常見的 Swagger 工具：

Swagger UI → 把 OpenAPI 文件轉成可互動的網頁
Swagger Editor → 線上編輯 OpenAPI 文件
Swagger Codegen → 依據 OpenAPI 產生客戶端 / 伺服端程式碼
Swashbuckle (ASP.NET Core 套件) → 自動把 Controller 轉成 OpenAPI

builder.Services.AddSwaggerGen();
👉 使用 Swashbuckle，幫你自動產生 OpenAPI 文件。
builder.Services.AddOpenApi();
👉 這是 ASP.NET Core 8 新的內建簡化方法，會幫你自動設定 OpenAPI 產生器，效果和 AddSwaggerGen() 類似，但更簡化。
👉 背後還是基於 OpenAPI (標準)，只是 .NET 8 開始有了官方支援。

為什麼需要 OpenAPI？

標準化 API 文件

不同語言 / 框架的 API，只要用 OpenAPI 格式描述，就能統一表示。

跨平台工具支援

前端 (TypeScript, Angular, React)、後端 (Java, Python, .NET)、測試工具 (Postman, Insomnia) 都能讀 OpenAPI。

自動化開發流程

可以用 OpenAPI 自動產生：

客戶端 SDK (C#, TypeScript, Java…)

伺服端樣板程式碼

文件網站

降低維護成本

後端修改 API → OpenAPI 自動更新 → 前端文件同步 → 減少「前後端溝通不一致」的問題。

技術上 可以並存，但通常沒有必要。

AddSwaggerGen() → /swagger/v1/swagger.json

AddOpenApi() → /openapi/v1.json
結果就是你的應用程式會同時提供兩份 API 文件。

👉 實務上建議：

簡單專案 / 學習：用 AddOpenApi() 就好。

正式專案 / 有認證、版本、客製需求：用 AddSwaggerGen()（Swashbuckle）。

Swashbuckle 和 NSwag

🔹 Swashbuckle

來源：社群套件，ASP.NET Core 官方文件推薦使用。

NuGet 套件：Swashbuckle.AspNetCore

主要用途：

自動產生 OpenAPI (/swagger/v1/swagger.json)

自動掛 Swagger UI (/swagger)

支援自訂 Schema、OperationFilter、文件版本化

自訂 Schema
用途：修改 Swagger 產生的模型描述（Schema）。
情境：你有一個敏感欄位，不想出現在 API 文件裡。
public class User
{
public int Id { get; set; }
public string UserName { get; set; }
public string PasswordHash { get; set; } // 不想暴露
}

SchemaFilter
using Microsoft.OpenApi.Models;
using Swashbuckle.AspNetCore.SwaggerGen;

public class HidePasswordSchemaFilter : ISchemaFilter
{
public void Apply(OpenApiSchema schema, SchemaFilterContext context)
{
if (context.Type == typeof(User))
{
schema.Properties.Remove(“passwordHash”);
}
}
}

builder.Services.AddSwaggerGen(c =>
{
c.SchemaFilter();
});

OperationFilter
用途：修改 API 動作 (Operation) 的 Swagger 描述。
情境：幫 API 加上 全域的 Header 說明，例如「必須帶 X-Request-Id」。

public class AddRequestIdHeaderFilter : IOperationFilter
{
public void Apply(OpenApiOperation operation, OperationFilterContext context)
{
operation.Parameters.Add(new OpenApiParameter
{
Name = “X-Request-Id”,
In = ParameterLocation.Header,
Required = true,
Description = “每個請求必須帶唯一 ID”,
Schema = new OpenApiSchema { Type = “string” }
});
}
}
builder.Services.AddSwaggerGen(c =>
{
c.OperationFilter();
});

文件版本化 (API Versioning)
用途：讓同一個 API 系統支援多個版本（例如 v1, v2），並在 Swagger 裡分開顯示。
情境：Todo API v1 只回傳標題，Todo API v2 回傳標題 + 完成狀態。

[ApiController]
[Route(“api/v{version:apiVersion}/[controller]”)]
[ApiVersion(“1.0”)]
[ApiVersion(“2.0”)]
public class TodoController : ControllerBase
{
[HttpGet, MapToApiVersion(“1.0”)]
public IActionResult GetV1() => Ok(new[] { new { Title = “Buy milk” } });

[HttpGet, MapToApiVersion("2.0")]
public IActionResult GetV2() => Ok(new[] { new { Title = "Buy milk", IsDone = false } });

}
註冊 API 版本化 + Swagger
builder.Services.AddApiVersioning();
builder.Services.AddVersionedApiExplorer(options =>
{
options.GroupNameFormat = “‘v’VVV”; // v1, v2
options.SubstituteApiVersionInUrl = true;
});

builder.Services.AddSwaggerGen(options =>
{
var provider = builder.Services.BuildServiceProvider()
.GetRequiredService();

foreach (var desc in provider.ApiVersionDescriptions)
{
    options.SwaggerDoc(desc.GroupName, new OpenApiInfo
    {
        Title = $"Todo API {desc.ApiVersion}",
        Version = desc.ApiVersion.ToString()
    });
}

});

Swagger UI 會有兩個文件：v1 和 v2。

GET /api/v1/todo → 只回傳 Title
GET /api/v2/todo → 回傳 Title + IsDone

SchemaFilter → 改模型描述（例：隱藏欄位、修改格式）

OperationFilter → 改 API 操作描述（例：全域加 Header、修改參數描述）

文件版本化 → 支援多 API 版本，Swagger UI 分組顯示

特色：

專注於 API → 文件（從程式碼產生文件）

ASP.NET Core 社群使用最廣，教學/範例最多

功能完整，支援多種安全性（JWT、OAuth2）

👉 最常見的選擇，如果只需要產生文件與 UI，Swashbuckle 就足夠。

🔹 NSwag

來源：由 Rico Suter 開發，功能比 Swashbuckle 更廣。

NuGet 套件：NSwag.AspNetCore

主要用途：

跟 Swashbuckle 一樣能產生 OpenAPI 文件和 Swagger UI

額外支援 API Client 產生器：可以根據 OpenAPI 產生 C# Client 或 TypeScript Client

可整合到 MSBuild / CLI / NPM，適合 CI/CD

特色：

不只是文件，還能做 API SDK 生成

適合前後端分離專案，前端可直接用 TypeScript Client SDK 呼叫 API

有 GUI 工具 NSwagStudio，方便點選操作

👉 如果專案需要「自動產生 API 客戶端」(C# 或 TypeScript)，NSwag 更適合。
