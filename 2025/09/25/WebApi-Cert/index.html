WebApi - Cert

Created2025-09-25|Updated2025-09-25|WebAPI
|Post Views:

憑證 = 網域的身分證
瀏覽器連到 https://example.com 時，伺服器會出示一張憑證，聲明：「我就是 example.com」。

憑證裡會包含：網域名稱 (CN / SAN)、發行者 (CA)、有效期限、公鑰等等。
CA = 證件簽發單位

瀏覽器或作業系統會檢查：這張憑證是不是由「我信任的單位」(受信任的 CA) 簽發的。

如果是 → 顯示安全連線 🔒

如果不是 → 出現警告（「此網站的憑證不受信任」）。

Root Certificate (根憑證)

你現在看到的提示，實際上是要安裝一張 本機開發環境自簽的 Root Certificate。

一旦安裝，代表：
👉 「我願意信任所有由這個 Root Certificate 簽發的憑證」。

所以以後即使是 https://localhost:5001 或 https://mytest.local，瀏覽器也會乖乖相信它安全。

localhost 憑證

不是公開 CA 發的，而是 你電腦自己產生的自簽憑證 (通常是 dotnet dev-certs https 或 Visual Studio 產生)。

意義是讓 Chrome / Edge / Firefox 這些瀏覽器，在你本機開發時，不會一直跳「不安全」。

✅ 沒錯，這個憑證就是讓瀏覽器「信任特定 domain（這裡是 localhost）」的。

它只在本機環境有用，不能放到正式環境（因為別人電腦不會信任你的 localhost 憑證）。
正式環境要用 公開 CA 發的憑證（例如 Let’s Encrypt）。

「localhost 憑證」就像自己畫的護照，正常國家不會承認，但你電腦自己說「我承認」，那就能用了。

https

HTTP vs HTTPS 差別

HTTP：明文傳輸，資料可能被攔截或竄改。

HTTPS：加上 TLS/SSL 加密，傳輸前會先交換憑證，建立安全通道。

憑證的作用（三件事）：

身份驗證 (Authentication) → 確認伺服器真的是它聲稱的 domain。

加密 (Encryption) → 建立 TLS 加密通道，避免資料被竊聽。

完整性 (Integrity) → 防止傳輸過程中被竄改。

📌 所以沒有憑證就不可能有 HTTPS。憑證就是 HTTPS 的核心。

公有 CA 發的憑證（正式用，像 Let’s Encrypt, DigiCert）
👉 瀏覽器/OS 預設信任，線上網站一定要用這種。

自簽憑證 (Self-signed certificate)（本機開發、測試用）
👉 例如你看到的 localhost 憑證。這種瀏覽器不會自動信任，要自己手動安裝。

開發憑證 (dotnet dev-certs)
👉 .NET Core / Visual Studio 自動產生的，目的是讓你在 https://localhost 測試時不會跳「不安全」。
