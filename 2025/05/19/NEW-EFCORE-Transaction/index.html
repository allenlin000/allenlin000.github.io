EF Core 更新資料時會怎麼包 Transaction？


Using Transactions

「交易（Transaction）」是用來確保一連串的動作只接受全部成功或全部失敗的一種保護機制。是一個「全部做完才算數」的保證。如果中間某一步出錯，你在畫畫，如果畫錯了，你要把整張圖擦掉重畫，而不是只改其中一筆錯的，讓整張圖不一致。交易就是這塊橡皮擦，它保證資料不會「一半對一半錯」。

SaveChanges() 自動包成 Transaction

SaveChanges() 會把你針對 DbContext 做的多個新增/修改/刪除，自動包在一個隱含的 Transaction 裡。如果有一筆操作失敗，整個操作會取消（Rollback），資料庫不會留下半套資料。

使用 BeginTransaction() 包多次 SaveChanges()

透過 DbContext.Database.BeginTransaction() 可取得 IDbContextTransaction，可將多次 SaveChanges() 包成一個 Transaction，由程式決定何時 Commit 或 Rollback。

有時你會在一個流程中呼叫多次 SaveChanges()，但希望它們全部算一筆交易。這時就要自己手動開一個 Transaction。

  • BeginTransaction()：啟動一個交易
  • Commit()：全部成功時提交
  • Rollback()：出錯時全部撤回

多個 DbContext 共用 Transaction

有時你會有兩個不同的 DbContext（比如一個管理會員資料、另一個處理訂單），但你希望這兩邊的資料更新動作一起成敗。所以你可以讓它們共用同一個資料庫連線（DbConnection） 和 同一個交易（Transaction）。

使用 TransactionScope 自動處理交易

TransactionScope 是一個更高層的做法，它會自動幫你處理交易的範圍。你只要把程式碼包在 using (var scope = new TransactionScope()) 裡，它會自動幫你記得要 Commit() 或 Rollback()（當然你得呼叫 scope.Complete() 才會成功）。
