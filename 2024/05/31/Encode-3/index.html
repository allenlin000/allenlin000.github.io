數位簽章(Digital Signature) ? Digest ?

曾經聽過 “中間人攻擊” 嗎?

當我們資料傳輸過程中，要怎麼驗證資料遭到竄改，這段資訊是不是真的來自發送方的本意，她是不是真的要跟我分手我不信等等
不過 現實中，人類就算面對面對談，有時候還是很難傳達本意，有時候比被竄改還麻煩呢，阿…離題了

總之，因為有驗證資料的需求，數位簽章就誕生了

但需要注意，他還是不能防止訊息被篡改，而是讓我們能夠判斷 “訊息是否在傳送過程中被改變了” 了?

舉例 假如我們今天一段訊息要傳送，簽章的過程是這樣的:

1.使用 hashing algorithm 做 Hash，例如：SHA-256、SHA-384、SHA-512 ，這些函數提供了不同的安全強度將這段訊息加以運算，而計算出的 Hash 也叫做 message digest，意義就是從內文摘要成一小片段的資訊

2.接著對這段 Digest 進行加密，於是簽章就做好了

3.當我們送出 Request 時，會把簽章與內文，一起送出，讓接收方可以透過自己再計算一次內文的Hash與 簽章簡密後的 Hash 做比對，來判斷資料是否有變化
而且這個 Hash 就算中間被劫持了，也無法被還原成原始資料（因為是單向的）。

但仍要注意，這一切的基礎還是私鑰需要保管好，若私鑰洩漏，簽章與加密什麼的都也沒有意義了，簽章只是一個機制告訴接收方，現在遇到的問題是”內文比對不一致!

另外，因為總覺得只因為想比對資料是否來自同一個來源就需要另外搞一個有點複雜的作法，那我直接對內文做非對稱加密應該也還是能讓接收者確定來源吧，畢竟私鑰公鑰是配對的

後來網路上再看了一些資訊後，發現了其他的好處，有人提到，因為數位簽章是只對長出來的 “Digest” ，也就是那一段 Hash 進行加密，因此 “效率” 會好得多
