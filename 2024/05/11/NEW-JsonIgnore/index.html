Json Ignore

Created2024-05-11|Updated2025-04-23|C# 舞想
|Post Views:

最近遇到一個有點詭異的 bug —— 前台文案怎麼樣都改不掉，一直顯示舊的內容。眼睛快脫窗、Redis 清了又清，才發現原來是某個屬性被掛上了 JsonIgnore。

即是說，這個屬性拿到了翻譯後的新文案，在存進 RedisCache 時，它根本就沒被序列化進去，結果下次重刷又抓到舊資料，簡直在演輪迴文案地獄。

其實我早就看到那行 JsonIgnore，但想說先放著不管它……。這大概就是所謂的：你不理它，它也不理你的某種境界。

順手搜了一下專案，這東西使用率還挺高的

Jason Feels Ignored

👾 JsonIgnore Attribute 有什麼樣的作用？

基本上在序列化和反序列化期間會忽略這該死的屬性，目的是資料不要傳遞出去，我們熟知的 Text.Json 以及老牌的 Json.NET 都有支援 JsonIgnore Attribute，作用上基本也相同。

舉個栗

假設有一間公司他的工作簡歷需要填上你是否是光頭的資訊，因為實在太不好意思，所以我只好偷偷加上 JsonIgnore 沙小

今天有人對我發出 API Request 時，送出去呈現出來的 json 資料將會忽略它!

👾 使用情境

  1. 不想暴露庫存細節給外人知道
  1. 計算屬性或冗餘資料

就是有先資料我們需要內部紀錄使用，但凡外人做 Request 不會看到這些秘密

👾 想更彈性點？來點動態 Ignore

某些時候不同角色需求不同，要忽略的屬性也不同，這時候硬寫死就太死板了。

幸好 System.Text.Json 支援客製化的 JsonTypeInfoResolver，讓我們能夠動態決定哪些屬性該藏起來。

👾 實例分享：Model Binding 會被影響嗎？

讓我們來聊聊這個 facebook 的情境

[JsonIgnore] 會影響模型繫結嗎？

看到一個討論：ASP.NET Core 中，如果你在 model 裡加了 [JsonIgnore]，那麼這個欄位在 model binding 階段也會跟著無視你。

也就是說，如果你有個密碼欄位：

加上 [JsonIgnore] 雖然能防止它被序列化出去，但同時也讓使用者從 Request Body 傳進來的值被忽略了。結果就是：後端根本拿不到密碼欄位。

解法：

  1. 分層處理

用 DTO + ViewModel 分開處理，乾淨、清楚，但程式碼稍多。

  1. 使用 Newtonsoft.Json

換成它的 [JsonIgnore] 就不會影響 model binding，兩邊各過各的生活。

想穩妥、可維護性高，選方案一
想快速解套、偷懶有理，選方案二（畢竟 Newtonsoft 是老司機）

👾 結語

就像你每天出門都告訴自己「鑰匙應該在包包裡」，但從沒確認過，直到某天站在門口發現自己被鎖在外面，才想起那句話其實只是有點懶

這些微不足道的忽略，看起來沒什麼，但累積起來就會變成你 debugging 時想砸螢幕的那顆地雷。

當你選擇忽略什麼的時候，也同時可能決定了你要承擔些什麼。

這不只是寫在屬性上的 JsonIgnore，更是我們寫在人生的 if-else 裡，每一次選擇的注解。
