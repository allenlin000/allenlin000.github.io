HttpClient (三. Refit )

Created2024-05-21|Updated2024-12-07|C# 舞想
|Post Views:

最近在專案中看到 Refit 這個套件的使用痕跡，好奇之下發現它使用它來做 HttpClient好像挺省事的

簡單來說，他的概念就是讓我們能夠以 Interface 來定義 怎麼串接第三方 API 。

所以我們要定義 Interface，Refit 會將 Interface 的方法包裝起來，處理 HTTP 請求並將 Response 資料序列化為 Interface 中指定的類型。

另外也可以組上自製的 HttpMessageHandler 以及處理Retry機制

…

讓我們逐一來看看實作吧!

Refit 的優勢之處

Type Safety

我們把方法簽章的概念套用在對第三方用 HttpRequest 上的一大好處就是，開發者想要叫用時，他必須傳入我規定的型別的參數，處理 Response 時也要用介面定義的型別處理，而且今天我們要查看每支 EndPoint 的規格，基本上去找 Interface 就可以一目了然了，開發者用錯時，在 compile time 就會噴錯

Automatic Serialization/Deserialization

另外你有發現有一件很煩的事情被解決惹嗎，我們不用自己去序列化 & 反序列化了，真是太屌了

支援基本上所有 Http 動詞

支援 .NET 框架的 DI (Dependency Injection) 或 直接使用

結語

有時遇到各式各樣的工具和套件，有些讓人眼前一亮，有些可能食之無味。Refit 這個小夥伴確實省下不少力氣，讓我們可以用更優雅的方式處理 API 串接。
不過套件終究是工具，重要的是了解它能解決什麼問題、帶來什麼便利。今天寫這篇文章，也是想跟大家分享這個發現，畢竟程式人生裡，少走點冤枉路也是一種幸福吧
