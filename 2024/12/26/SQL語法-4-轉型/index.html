CAST and CONVERT 差異是甚麼 ?

參考文件 : https://learn.microsoft.com/en-us/sql/t-sql/functions/cast-and-convert-transact-sql?view=sql-server-ver16

CAST

  • ANSI SQL 標準的一部分，考慮可移植性（如 Oracle、PostgreSQL、MySQL 等）可以使用
  • 語法簡單，常用於通用轉型。
  • 語法例子：CAST(expression AS target_data_type)

CONVERT

  • 是 Microsoft SQL Server 的擴展語法，不是 ANSI 標準的一部分，因此僅適用於 SQL Server。
  • 提供更多格式選項，特別是對於日期和時間的格式化。
  • 語法範例：CONVERT(target_data_type, expression, style)

因此，CAST 偏向遵循通則的語法，而 Convert 在 MSSQL 中對於日期格式的調整較為便利

CONVERT 和 FORMAT

格式轉換時，也滿有機會看到 FORMAT 的使用，比較一下他們的特點

FORMAT

  • 是 CLR（Common Language Runtime）函數，基於 .NET Framework。
  • 用於靈活格式化資料，支持多種文化（如地區語言）。
  • 性能較差，因為它需要執行更複雜的字符串操作。

特點是依賴 CLR，需要將資料從 SQL Server 引擎轉移到 CLR 執行，增加了上下文切換的開銷，導致性能下降。

Convert

  • 是 SQL Server 原生的函數，用於數據類型之間的轉換（如日期轉字符串，字符串轉數字等）。
  • 相對來說性能高，專為處理數據轉型設計。
  • 提供有限的格式化能力（例如日期的樣式代碼）。
