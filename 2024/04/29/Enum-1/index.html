Enum (一. 探討 Enum Value 的使用)

Created2024-04-29|Updated2024-12-30|C# 舞想
|Post Views:

Enum 是一種「帶有名稱的整數集合」

就像點餐的菜單，裡面有「漢堡、薯條、可樂」三個選項。
每個選項其實都有一個編號，比如：

漢堡是 1
薯條是 2
可樂是 3

你點餐時說「我要薯條」，程式會把「薯條」記錄為 2。
對程式而言，他是 2 ，對人類而言，他是「薯條」

我覺得 Enum 有點像機器語言、程式語言、自然語言之間的關係

狀態、開關控制、選項

訂單的狀態（待處理、處理中、完成、取消）

在城市裏面我們可以選擇在需要的地方寫死，我們今天可以根據不同的訂單狀態做出對應的處裡可以這樣寫

但問題就在於，這種寫死的作法很容易寫錯，並且類似的意義每個開發者可能用的文字不一樣，程式碼會很混亂各自為政

更好的做法是定義一組訂單狀態列表，大家遵循或修改都依照這個列表

Flags Enum

Flags Enum 是一種特殊的 Enum，它用來表示一組可能的位元組合（bitwise combinations），適合用來 “定義多個可以同時啟用的選項”

普通的 Enum 通常是用來表示互斥的狀態，例如交通燈的紅、黃、綠。但在某些情況下，你可能需要一個物件同時具有多個屬性。例如：

檔案權限（可以同時讀取與寫入）
按鈕的外觀（同時啟用粗體、斜體）

Flags Enum 的背後原理就是用 ** 位元運算（bitwise operations）** 來組合多個選項。

假設你在玩角色扮演遊戲（RPG），你的角色可以有多種技能，比如「隱身」、「飛行」和「射箭」：

普通的 Enum 只能選一種技能。
Flags Enum 讓你同時擁有多種技能，像是在一張技能表上打勾。

定義檔案的權限

檔案可能有以下幾種權限：

讀取（Read）
寫入（Write）
執行（Execute）

有時候檔案可能同時有「讀取」和「寫入」的權限。

或是

使用 Flags Enum 的注意事項

  1. 為每個選項指定 2 的次方值
    Flags Enum 的核心是二進位運算，所以每個值必須是 2 的次方（例如：1, 2, 4, 8, 16）。這樣才能確保每個選項在位元層級上是唯一的。

  2. 加上 [Flags] 特性
    雖然 Flags Enum 沒有強制要求 [Flags] 特性，但加上可以讓程式更加語意化，而且能改善 ToString() 的行為。

Description

在實際開發中，經常會遇到需要讓 Enum 不僅表示一個值（如整數或名稱）
還需要對應一段更具描述性的資訊。這種需求可以透過在 Enum 上掛 Description 屬性來實現。

  • 顯示在 UI 上的內容。
  • 記錄在日誌中的資訊。
  • 提供更多語意化的上下文。

JSON 序列化與反序列化

[JsonConverter(typeof(StringEnumConverter))] 是在 .NET 的 JSON 序列化與反序列化 過程中，控制 Enum 如何被轉換的一種方式，主要使用於 Newtonsoft.Json。

  1. 將 Enum 轉為字串： 當序列化（Serialize）時，Enum 的值將轉換為對應的名稱（字串）而不是整數。
  2. 將字串轉為 Enum： 當反序列化（Deserialize）時，字串會被轉換為對應的 Enum 值。
