SQL Injection

Created2024-09-22|Updated2025-05-22|Others
|Post Views:

✨ ① FromSqlRaw(“EXEC … {0}”, user)

  • {0} 是 EF Core 提供的「格式化佔位符」。
  • 傳入的 user 會變成內部的 @p0 參數，交給資料庫執行。
  • ✅ 安全、簡潔。
  • ❗ 當 SQL 太複雜（多個參數、多行）時，閱讀性下降。

✨ ② FromSqlInterpolated($”EXEC … {user}”)

  • 背後使用 FormattableString，會自動辨識 {user} 為參數，產生 @p0。
  • ✅ 同樣安全，而且語法非常直覺、可讀性最佳。
  • 📌 不支援複雜的 T-SQL 多行語法時可能會變得難以維護（但大多數情況都夠用）。
  • ✅ 建議做法：如果沒有特殊需求，這是最推薦用法！

✨ ③ FromSqlRaw(“EXEC … @user”, new SqlParameter(…))

  • SqlParameter 是 ADO.NET 的參數物件。
  • 可以自訂資料型別、長度、方向（IN / OUT）、Nullable 等。
  • 適合你需要更多控制，例如：
    • 使用 SqlDbType
    • Output 參數
    • Nullable 明確控制
使用情境 建議用法
想寫得直覺、清楚好讀 FromSqlInterpolated(...) ✅
傳入多個參數、格式佔位符需要控制順序 FromSqlRaw("... {0}, {1}", p0, p1)
需要細節控制（型別、方向、長度） FromSqlRaw(...) + SqlParameter
