泛型委派

事實上這三個委派可能僅是在傳入以及回傳型別不同，我們直接定義成一個參數的感覺

其實這就是所謂的泛型

稍微複習一下泛型

它是在 .Net 2.0 新增的功能。泛型將型別參數的概念引進 .NET Framework 中，使得類別和方法在設計時，可以先行擱置一或多個類型的規格，直到用戶端程式碼對類別或方法進行宣告或具現化時再行處理。

泛型在參考型別的運作方式稍有不同。第一次搭配任何參考型別建構泛型型別時，執行階段會建立特定的泛型型別，以物件參考取代 MSIL 中的參數。然後建構類型每次搭配參考型別具現化為其參數時，不論類型為何，執行階段都會重複使用先前建立的特定版本泛型型別。

什麼意思呢?

  1. value type 和 reference type 的主要差異在於 JIT 編譯時的行為
  2. value type 會為每種類型生成專門的程式碼
  3. reference type 會共用相同的程式碼實作
