Shallow Copy & Deep Clone

Created2024-11-03|Updated2025-04-09|Others
|Post Views:

在我們撰寫程式邏輯時，常會碰到傳參問題，傳參就是傳送參數的簡稱 (其實可能只有我這樣簡稱辣)，當我們要把一個 Entity 寶寶傳來傳去做一些操作時，我們都要想到，我會不會改動到原始的內容？還是我只是想要複製一份出來？

地鳴寶寶

Shallow Copy & Deep Copy


source : algodaily.com

左圖表示，我們實例化兩個物件，stack 都是 reference 到相同的記憶體位址
右圖是，Obj2 Copy 自 Obj1，連記憶體位址都獨立出來了

也就是說，操作物件 Obj1 並不會干涉 Obj2

當我們想自己決定要不要被改名叫比爾時完全複製時，我們需要了解有哪些作法來處理這個問題

IClonable

某次在專案中看到有人實作了 IClonable，好奇之下看了文件

他只叫你實作 Clone 方法，至於他是 Deep Copy 還是 Shallow Copy…沒人定義
這樣感覺自己定義個 public Shallow Copy, public DeepCopy 是不是會比較清楚啊 🦥
我個人仍不確定使用這個介面的必要性
