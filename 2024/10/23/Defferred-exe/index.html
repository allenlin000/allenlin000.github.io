結果

當我們使用 yield return 時，Compiler 會自動為我們生成一個實作 IEnumerator 的類別，這個類別包含了 MoveNext 的實作，原始碼會變這樣

所以當我們執行

  1. city.GetEnumerator() 返回編譯器生成的 Iterator class 實例
  2. 這個生成的類別有完整的 MoveNext/Current 實作
  3. foreach 使用這個生成的迭代器來走訪集合

因此這個走訪過程是

  1. 進入 foreach
  2. 執行 GetEnumerator () , 得到一個 IEnumerator
  3. 執行 MoveNext () , 以判斷走訪是否結束。若尚未結束則將 Current 屬性移動到下一個元素
  4. 回傳 Current 屬性給 item

不論是 IEnumerable 或是 IEnumerable 都提供一個 GetEnumerator () 方法。再透過所得到的 Enumerator 物件去執行走訪這個動作

Deferred Execution 是怎麼回事?

直觀來說，程式執行到哪一行，那一行就會被執行。但 LINQ 有一個很重要的特性，叫做「延遲執行」（deferred execution）, 或稱為惰性求值（lazy evaluation）. 也就是叫他忍住，在需要取用查詢結果的時候，才去執行查詢表示式.

WHERE 不也是 Yield Return 嗎，與 IQueryable 操作資料庫的延遲執行差異是什麼?

IEnumerable.Where()，內部確實有實作 yield return，但這是在記憶體內部操作的， 撈資料庫時，會把資料都撈出來才在記憶體做篩選行為(LINQ to Objects)，而 IQueryable.Where()則是轉換成 SQL WHERE 子句，也就是會在資料庫層面篩選，只傳回需要的資料，二者不同

  • Multiple Enumerations: If you enumerate over a deferred execution query multiple times, the query will be executed multiple times, which can lead to performance issues.

  • Side Effects: If the data source changes between enumerations, the results may differ each time you iterate over the query.

  • Captured Variables: Be cautious when using variables captured by a lambda expression in a deferred query. If the value of the variable changes before the query is executed, the new value will be used.
