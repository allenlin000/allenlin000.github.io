簡介

使用 Thread 類別的 QueueUserWorkItem 方法來進行非同步操作缺了兩項基本功能：

  1. 得知非同步工作何時完成
  2. 當工作完成時取得其執行結果

微軟從 .NET Framework 4 開始提供了以 Task 為基礎的（task-based）概念，並將相關類別放在 System.Threading.Tasks namespace 裡。這些 Class 形成了一組 API，統稱為 Task Parallel Library，簡稱 TPL。到了 .NET Framework 4.5，TPL 又有一些擴充和改進，程式寫起來又稍微容易些。
