HttpContext

Created2024-06-27|Updated2024-06-30|HttpContext
|Post Views:

為了理解什麼是 HttpConext，我們首先想要知道 Context 是甚麼，中文翻作 “上下文”

例如，有一天 毛吉對著阿冠說 :「給我100元」，對於沒頭沒尾，沒有 “上下文” 的一句話，雖然看到毛吉毛茸茸的不免產生憐愛之心，但理性的阿冠，不明不白的當然不會答應，只能搖搖頭想說，這神經病想幹嘛

於是開始學習溝通的技巧，閱讀了腦袋的部落格，理解了甚麼是上下文的概念，他再次跑去找阿冠說 : 「阿冠美女，我是腦袋最愛的小狗勾，我肚子好餓，可以給我100元嗎」，阿冠這時候，得到了這個 “場景” 的 “更多訊息” (有禮貌、腦袋的狗、肚子餓)，於是阿冠擁有更多的資訊可以判斷，自然也能知道做出什麼對應的行為了 !

Context 一詞核心理念是提供一個環境或背景，允許理解和操作特定的資訊。在程式設計中，上下文通常指的是在特定時間點，與某個操作或事件相關的所有狀態和環境信息。

回到 HttpContext，當一個 HTTP Request 來到 我們的程式中，Kestrel 會接收這個請求並將它封裝為一個 HttpContext Class，然後將 HttpContext 傳遞給 ASP.NET Core 的 Middleware，Request 在我們的程式一段段執行往下傳遞時，我們可以藉由這個包裝好的 HttpContext，也就是這個 “Http Request 的 上下文” 得到這個發出者的訊息，包含 URL 路徑 QueryString、Header、Identity… 做出對應的檢查與操作

** Kestrel 是 ASP.NET Core 的內建 Web 伺服器，也就是說，它是 ASP.NET Core 框架的一部分，當你建立並執行一個 ASP.NET Core APP 時，它會自動使用 Kestrel 來處理 HTTP 請求，而 HttpContext 則是 ASP.NET Core 中的一個資訊載體，用於表示當前 HTTP Request 的上下文。
