Query Expression Syntax vs Method Chaining Syntax

Created2024-06-06|Updated2025-01-01|C# 舞想
|Post Views:

在 .NET 中，Query Syntax 和 Method Chaining 是兩種常用的方式來處理資料查詢，特別是在使用 LINQ（Language Integrated Query）時。這兩者本質上是做相同的事情，只是表達方式不同

什麼是 Query Syntax？

Query Syntax 是一種類似 SQL 的語法，使用關鍵字來撰寫查詢。這種語法比較接近人類語言，容易讓人理解查詢的邏輯。
想像你在用問句來挑出玩具，比如：「哪一些玩具是紅色的？」你先描述條件，然後說明你想要什麼結果。

這裡的 from、where 和 select 就像 SQL 的語法，讀起來很直觀。

什麼是 Method Chaining？

Method Chaining 是使用方法（Method）來連續操作資料的語法風格。每個方法都會回傳一個結果，然後下一個方法繼續處理這個結果。
想像你有一個玩具清單，先挑出紅色的，然後把它們按大小排序，最後只拿出最小的那個。每一步都像在給玩具「過濾」或「排序」。

性能

延遲持行

Query Syntax 和 Method Chaining 的運行效率本質上是相同的，它們最終都會被編譯成相同的 IL (Intermediate Language) ，並由 LINQ 提供的查詢運算子處理資料

不管是 Query Syntax 還是 Method Chaining，LINQ 的查詢運算都是 延遲執行 (Deferred Execution) 的。意思是，查詢本身只是設定了邏輯，直到你真的「迭代」（比如 foreach 或 ToList()）時，才會執行過濾、排序等操作。

不論是 Query Syntax 還是 Method Chaining，它們背後用的是 LINQ 的運算子（如 Where、Select 等），執行時會進行一個「每次一個項目」的處理，避免對整個集合進行不必要的操作。

使用時機

實驗環境

LINQPad 提供了一個方便的環境，讓我們可以同時編寫查詢語法和方法呼叫，並即時查看它們之間的轉換。

差異

Query Syntax 為包含所有運算方式

Method syntax 中可用的標準查詢運算子，有部分並未包含在 Query Syntax 中，例如：Distinct、Take 等等，若要使用這些運算子，就必須使用 Method syntax 來撰寫。

拆解性

Method syntax 讓我們可以將查詢分解成多個步驟，每個步驟都返回一個中間結果（IEnumerable），這樣可以更容易地組合和重用查詢的不同部分。
查詢語法通常是一個完整的查詢表達式，不太容易將查詢拆分成可重用的部分。

還有 let 的使用

假設我們有一個 Character 類別，表示遊戲中的角色，其中包含角色的名稱、等級和經驗值。我們想要查詢所有達到 5 級及以上的角色，並計算他們升到下一級所需的經驗值。

let 的作用是，讓我們可以暫時儲存運算後的資料作為後面的查詢使用

透過這個例子，我們看到 Query Syntax 中的 let 子句和 Method Syntax 中的 Select 方法結合匿名類型可以達到類似的效果。Query Syntax 使用 let 子句引入一個新的範圍變量，而 Method Syntax 則使用匿名類型的屬性來表示中間計算結果。如果理解 let 的用途的話， QUery Syntax 會比較好讀一些

從不同資料來源組資料時

這個與 JOIN 不同的是，我們可能不是要交集篩選他們的資料，而是單純從不同資料來源，列出不同的排列組合
