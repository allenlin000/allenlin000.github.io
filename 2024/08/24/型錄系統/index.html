Content Delivery Network（CDN）

Created2024-08-24|Updated2025-10-05|落葉下的存檔
|Post Views:

後端 API 流量也透過 CloudFront 分流

使用者打 API → 不是直接進 WebAPI 主機 → 而是先進 CloudFront，再由 CloudFront 轉給 Origin（真正伺服器）

  • 加速 API 回應（雖然 API 通常是動態，但部分回應仍可快取）
  • 防止主機直接暴露（安全）
  • 可控流量與限流
  • 統一流量進出口（方便監控）

現在 Referrer 發出的 API 請求會先打到 CDN，CDN（例如 d9c…cloudfront.net）會先判斷：

如果快取裡有資料 → 直接回應；
沒有才轉發給 Origin。

✅ Origin 流量大幅下降
✅ 回應速度變快（CDN 靠近使用者）
✅ 減少跨區延遲與成本

Resource ID

CloudFront 的唯一識別碼（例如 E10FW360Y04PQ3）。這是 AWS 為每個 CloudFront Distribution 分配的「ID」。在 AWS Console → CloudFront 裡面會看到這個欄位。這一欄是系統層級的「代號」，不是域名。

CNAME

就是這個 CloudFront Distribution 綁定的「自訂網域」
例如：

webapi.91mai.com
mobile.91app.io

意思是使用者實際打的 API 網域。CloudFront 會根據這個 CNAME 接住請求。

📘 小知識：
CloudFront 原始域名長得像 d9cwh2l3co8h5.cloudfront.net，
但我們會「綁一個自家網域」在前面（CNAME），
讓 API 網域更一致、可控。

CloudFront CNAME / Domain Name

這是 AWS 分配的「CloudFront 網域名稱」。通常格式是：xxxxxxxxxxxx.cloudfront.net
每個 Distribution 都會有一個。
用來測試 CDN 是否正常、驗證 CNAME 對應，或在緊急時可以直接用這個測試

原 Origin 設定

表示這個 CloudFront 背後實際對應的「原始伺服器」（Origin）
通常會是：

  • WebAPI 主機（自有 API）
  • AWS Load Balancer（ELB）
  • S3 或 其他 CDN 節點

Hit Rate 較低的 API 分析

/webapi/VIPMember/GetVipMemberCustomLinkSettings Hit Rate 低原因: 帶語系

個人化資料限流

個人化資料＝跟「某個使用者本人」有關、不能公開快取在 CDN 的資料
（例：我的收藏、購物車、推薦清單、訂單紀錄、個人優惠）。

因為這些請求不能像圖片/CSS 一樣丟給 CDN，通常都會直打你的 Origin（應用程式 + 資料庫）。
一旦有人（或機器人）短時間狂打「個人化 API」，你的 DB、應用程式就會被拖垮。

個人化資料限流（Rate Limiting） 的本質就是在入口處幫每個人裝一個「流量節流閥」：

同一個使用者、同一個 API，在一段時間內最多只能打幾次。

超過就回 429 Too Many Requests，或延遲、排隊。

目標：保護核心系統、維持穩定、公平使用、降低成本、避免資料爭用。

一句話：因為個人化不能靠 CDN 分擔，所以一定要靠限流 + 近源快取守住 Origin

判斷是否登入分流

判斷是否登入分流＝在請求一進來時，先確認「這個人是匿名訪客還是已登入的會員」，然後走不同的資料路線（Routing）與處理策略：

未登入（Anonymous）
多半只需要公開內容（商品清單、首頁、分類、熱門榜…）。
👉 可以大膽用 CDN 快取、Server 端 Output Cache、Aggressive Cache-Control，走最便宜最快的路。

已登入（Authenticated）
需要個人化內容（購物車、收藏、我的優惠、個人推薦）。
👉 不能公開快取，必須進應用程式與資料庫，走安全但較貴的路，搭配細緻的 Rate Limiting、微快取（Redis / Memory）、權限驗證（Authorization）。

本質：把「可廣播的」與「只屬於你的」請求分開。
目的：速度更快（匿名端）、資料更安全（會員端）、成本更低、系統更穩。

功能斷路（Feature Circuit Breaker）

在系統過載、異常、或流量高峰時，可以「局部關閉一些非關鍵功能」 來保護整體穩定

PageView

商品頁在載入時，會觸發的「行為追蹤、瀏覽記錄上報」功能
每次有人開「商品頁」，系統會去呼叫一個「記錄瀏覽事件」的 API
但這個 API 並不是用來顯示畫面的「主要功能」，它只是輔助性的「行為紀錄」或「推薦系統的資料來源」。

其他功能

  • 關閉收藏清單數量
  • 關閉購物車數量
